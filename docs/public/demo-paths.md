# Demo Paths

This note gives short demonstration routes through Antica Fiamma.

They are written for a first meeting or quick project review, giving visitors a guided path through the structure of the interface.

## Path 1: From Line To Commentary

Start:

- [Antica Fiamma](https://anticafiamma.it)

Route:

- choose a cantica and canto
- select a line in the Main Entry
- read the Analysis Layer / Line Snapshot
- open Close Reading
- inspect commentary cards, dates, sources, and full-text expansion
- pin selected cards into Compare

What this path demonstrates:

- the project begins from Dante's line
- commentary records remain tied to local textual context
- comparison is a reading act, not a detached search result

## Path 2: From Dante Word To Word Locus

Start:

- [Interface Tour](https://anticafiamma.it/reading-route.html)

Route:

- follow the Dante Word Locus Layer section
- open a line with selectable content words
- select a Dante word
- inspect Occurrence Explorer
- inspect Weighted Micro-Context Concurrence
- inspect Exact Local Phrase Expansions
- inspect Contrastive Interpretive Vocabulary

What this path demonstrates:

- choosing a word and choosing a line are different acts
- lexical recurrence can lead back into interpretation
- word-level navigation is tied to local commentary pressure rather than isolated frequency counts

## Path 3: From Research Room Back To Reading Interface

Start:

- [Fiamma Research Room](https://anticafiamma.it/research/fiamma.html)

Route:

- read the fire-motif framing
- inspect the visualizations and public research assets
- follow line-level references back toward the main interface
- compare what the motif-level view shows with what local commentary cards show

What this path demonstrates:

- research rooms are not detached essays
- a topic such as fire can become an interface path
- motif-level exploration should remain accountable to line-level reading

## Path 4: From Commentary To Authority

Start:

- [Authority Layer](https://anticafiamma.it/authority.html)

Route:

- inspect the public authority page
- open selected author, work, or personaggio rooms
- return to commentary records where authority references appear

What this path demonstrates:

- the commentary tradition is also a field of authors, works, figures, and inherited references
- authority navigation can be made public through static rooms
- authority pages supplement line-level reading rather than replacing it

## Path 5: Reading The Repository

Start:

- [GitHub public repository](https://github.com/ruocisong/ddp-workbench-public)

Route:

- read the root README
- read [project-dossier.md](./project-dossier.md)
- read [rights-and-permissions.md](./rights-and-permissions.md)
- read [technical-overview.md](./technical-overview.md)
- read [sustainability-and-maintenance.md](./sustainability-and-maintenance.md)

What this path demonstrates:

- the project has a public documentation layer
- the source archive and interface layer are separated
- the repository is a public shell, not the full internal workspace
