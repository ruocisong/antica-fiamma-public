# For Reviewers

This short path is for readers who want to assess **Antica Fiamma** quickly before a meeting or discussion.

## If You Have Ten Minutes

Read in this order:

- [scheda-progetto.md](./scheda-progetto.md): one-page Italian project sheet.
- [project-dossier.md](./project-dossier.md): compact project dossier.
- [rights-and-permissions.md](./rights-and-permissions.md): source, permission status, and reduced-version posture.
- [technical-overview.md](./technical-overview.md): technological and implementative choices.
- [sustainability-and-maintenance.md](./sustainability-and-maintenance.md): current maintenance model and hosting boundary.
- [demo-paths.md](./demo-paths.md): quick paths through the live interface.

## If You Have Two Minutes

Start with:

- [scheda-progetto.md](./scheda-progetto.md)
- [Fiamma Research Room](https://anticafiamma.it/research/fiamma.html)
- [Interface Tour](https://anticafiamma.it/reading-route.html)

## What This Set Of Documents Shows

Together, these documents show:

- the scholarly premise of the project
- the distinction between DDP as source archive and Antica Fiamma as interface layer
- the public shell / runtime data boundary
- the current rights and permission posture
- the technical structure of the front end, build scripts, deployment, and smoke checks
- the current author-maintained hosting and maintenance model
- the reduced-version path if modern commentary material must be removed

The aim is to make the project inspectable without requiring a reader to reconstruct its structure from the file tree.
