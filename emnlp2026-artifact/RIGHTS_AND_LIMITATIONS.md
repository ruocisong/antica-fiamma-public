# Rights and Limitations

Antica Fiamma should be read as a source-accountable workbench, not as an unrestricted redistribution of commentary text.

## Public Artifact Boundary

The public repository may include:

- static shell code;
- public documentation;
- selected build and runtime-check scripts;
- schemas and derived metadata where release is permitted;
- screenshots and rights-safe examples;
- small sample structures sufficient to understand the artifact.

The public repository does not promise full redistribution of generated commentary payloads. Full source-derived payloads remain governed by their source reuse conditions.

## Source Relationship

Antica Fiamma is built around commentary records associated with the Dartmouth Dante Project and related source structures. The live system preserves source links and distinguishes the source archive from the Antica Fiamma interface layer.

## Evaluation Limitations

- No human-subject evaluation is claimed.
- The line-record audit checks sampled source identity and line-span grounding; it is not a recall estimate.
- The authority consistency audit checks sampled runtime mentions with automatic predicates; it is not an extraction precision estimate.
- The case study illustrates workflow behavior; it does not measure user preference or task speed.
- Latency is reported according to the actual measurement mode used by the evaluation artifact.
